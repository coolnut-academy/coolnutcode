AI Coding Workflow For Thai Builders

ใช้ Claude, Codex, Gemini และ IDE ให้เป็นขั้นตอนเดียวกัน

หน้านี้อธิบายว่าแต่ละโหมดใน COOLNUTCODE ใช้เมื่อไร, output ส่วนไหนต้อง copy ไปวางที่ไหน, และควรทำอะไรต่อใน Claude Pro, VS Code + Codex, terminal, GitHub Pages หรือ Vercel เพื่อให้คนที่ไม่คุ้นกับ AI coding ก็เดินงานได้อย่างมั่นใจ

ภาพรวมแบบสั้น

1. คิดไอเดียในเว็บนี้

พิมพ์ไอเดียเป็นภาษาไทยใน Build From Scratch หรือเลือกโหมดเฉพาะงาน เช่น debug, patch, QA.

2. เอา output ไปวางในเครื่องมือที่ถูกต้อง

Claude เหมาะกับวางแผนและ review. Codex ใน VS Code เหมาะกับแก้โค้ดจริง. Terminal ใช้ตรวจ build/lint/test.

3. ทำทีละ phase

อย่าสั่งให้ agent ทำทั้งระบบในครั้งเดียว โดยเฉพาะงาน login, database, upload, payment หรือ production.

เริ่มโปรเจกต์ใหม่ด้วย Build From Scratch

1

ใส่ไอเดียหลัก

ตัวอย่าง: เว็บคำนวณค่า BMI หน้าเดียว หรือ ระบบ portfolio นักเรียน มี login dashboard admin นักเรียนอัปโหลดงานได้.

2

อ่าน Analysis และ Recommended Stack

ถ้าเป็น Level 1 มักทำ static ได้. ถ้าเป็น Level 3-4 ควรใช้ Claude วางแผนก่อน แล้วค่อยให้ Codex ทำทีละ phase.

3

Copy Claude Planning Prompt

นำส่วน Claude Planning Prompt ไปวางใน Claude Pro เพื่อให้ Claude สร้างไฟล์ planning เช่น 00_PROJECT_BRIEF.md ถึง 09_HANDOFF_SUMMARY.md.

4

สร้าง project-docs ใน VS Code

เปิดโปรเจกต์ใน VS Code แล้วสร้างโฟลเดอร์ project-docs. นำเนื้อหาที่ Claude สร้างมาใส่เป็นไฟล์ Markdown ตามชื่อที่กำหนด.

5

Copy Codex Implementation Prompt

นำส่วน Codex Implementation Prompt ไปวางใน Codex ใน VS Code. ให้ Codex อ่าน /project-docs/*.md และทำเฉพาะ Phase 1.

6

ตรวจด้วย terminal

หลัง Codex ทำเสร็จ ให้รันคำสั่งที่ phase ระบุ เช่น npm run build, npm run lint, หรือเปิดไฟล์ HTML ตรวจเองถ้าเป็น static.

เลือก IDE และเครื่องมืออย่างไร

Claude Pro

  • เหมาะกับวางแผน architecture
  • เหมาะกับ review diff และหา root cause
  • ไม่ควรให้ Claude เขียนไฟล์แทน Codex ถ้าไฟล์อยู่ใน VS Code

VS Code + Codex

  • เหมาะกับอ่านไฟล์จริงและแก้โค้ดจริง
  • ควรสั่งทีละ phase หรือทีละ bug
  • ควรให้ Codex รายงาน changed files และ commands ที่รัน

Terminal

  • ใช้รัน build/lint/test
  • copy error log กลับไปให้ Codex หรือ Claude
  • อย่าแก้ต่อถ้า command fail โดยไม่เข้าใจสาเหตุ

Stack ที่ระบบแนะนำหมายถึงอะไร

A. Static

index.html + Tailwind CDN + GitHub Pages เหมาะกับเว็บหน้าเดียว เครื่องคิดเลข prompt helper หรือ landing page. ไม่ควรมี login/database/secret.

B. Vite React

เหมาะกับหลาย component, state เยอะ, search/filter/export แต่ยังไม่มี auth หรือ database. ใช้เมื่อ static เริ่มดูแลยาก.

C. Next.js + Firebase

เหมาะกับ login, dashboard, role, upload, user data, permissions และ production deployment. ต้องระวัง security rules มากเป็นพิเศษ.

กฎสำคัญสำหรับผู้ใช้ทั่วไป

  1. ถ้า output บอกว่า Level 3 หรือ Level 4 ให้ใช้ Claude วางแผนก่อนเสมอ
  2. อย่าให้ Codex ทำทุก phase พร้อมกัน ให้ทำ Phase 1 เท่านั้นก่อน
  3. ถ้า Codex แก้ bug เดิมพลาดสองครั้ง ให้หยุด แล้วส่ง error log ให้ Claude วิเคราะห์ root cause
  4. ถ้าใช้ Firebase ต้องตรวจ security rules ก่อน deploy
  5. ถ้าเป็น GitHub Pages/static ห้ามฝัง API key หรือ secret ลงไฟล์เว็บ

COOLNUTCODE เป็นเว็บ static ที่เปิดบน GitHub Pages ได้ หน้า guide นี้ไม่ต้องใช้ backend และใช้งานคู่กับ หน้าเครื่องมือหลัก.
